# Шаг 1.12 — Обновить `.env.example` (Docker-порты)

**Этап:** 1. Docker-инфраструктура  
**Статус:** [x] Выполнен

## Описание

Добавить переменные окружения для Docker-портов в `.env.example`. Эти переменные используются в `docker-compose.yml` для проброса портов на хост.

## Что добавить в `.env.example`

В конец файла:

```env
# Docker ports (host)
APP_HTTP_PORT=28080
MYSQL_HOST_PORT=23306
REDIS_HOST_PORT=26379
```

## Таблица портов

| Переменная | Значение | Сервис | Внутренний порт |
|---|---|---|---|
| `APP_HTTP_PORT` | 28080 | nginx (HTTP) | 80 |
| `MYSQL_HOST_PORT` | 23306 | MySQL | 3306 |
| `REDIS_HOST_PORT` | 26379 | Redis | 6379 |

## Важно

- Порты выбраны так, чтобы **не конфликтовать** с другими проектами на хосте (см. таблицу занятых портов в BasePlan)
- После добавления — скопировать `.env.example` в `.env` и заполнить актуальные значения

## Зависимости

- Нет (файл `.env.example` уже существует)

## Критерий завершения

Переменные `APP_HTTP_PORT`, `MYSQL_HOST_PORT`, `REDIS_HOST_PORT` присутствуют в `.env.example`.
