# Шаг 13 — Тесты

## Цель

Подтвердить: миграции применяются, связи корректны, сидеры и импорт идемпотентны.
Тесты в `tests/Feature` (неймспейс `Tests\Feature`), PHPUnit `^12`.

## Набор тестов

### 1. Миграции применяются
- С `RefreshDatabase` все миграции прогоняются без ошибок.
- Через `Schema::hasTable()` проверить наличие ключевых таблиц
  (`contacts`, `banners`, `shipments`, `products`, справочники, pivot'ы).

### 2. Связи моделей
- Создать `Product` с фабрикой/вручную; привязать `category`, `tags`, `regions`,
  `images`, контентные/статусные блоки.
- Проверить, что связи возвращают ожидаемые типы и количества.
- Проверить cascade: удаление `Product` удаляет `product_images`, контентные/статусные блоки, pivot-строки.
- Проверить `nullOnDelete`: удаление `Category` обнуляет `products.category_id`.

### 3. Уникальные индексы
- Повторная вставка `external_id` / `sku` / `slug` / `tags.name` бросает исключение.
- Дубликат пары в pivot (`product_tag`, `product_region`) запрещён.

### 4. Сидеры идемпотентны
- Двойной запуск сидеров не увеличивает число строк в справочниках.

### 5. Импорт XML идемпотентен (ключевой тест)
- Подготовить фикстуру-XML (минимальный валидный фрагмент `advertisements_export`).
- Прогнать импорт дважды; убедиться, что число `products` и связанных записей не меняется
  после второго прогона (upsert по `external_id`).
- Проверить корректность маппинга нескольких полей (`name`, `price`, `is_main` у изображения,
  `product_region`, `product_manager.role`).

## Запуск

```bash
php artisan test
# или конкретные группы
php artisan test --filter=Catalog
```

> Для тестовой БД использовать конфигурацию из `phpunit.xml`. Фикстуру XML положить в
> `tests/Fixtures/` (создать директорию).
